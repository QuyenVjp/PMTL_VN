# PMTL UI/UX Specialist

Legacy compatibility skill. Prefer `pmtl-ui-behavior` unless the prompt explicitly names this specialist pass.

Use this skill for the “how it feels” layer of the product.

## Workflow

1. Identify the interaction surface.
2. Decide which behavior family matters most:
   - forms and validation
   - navigation and focus
   - loading/empty/error/success states
3. Read only the relevant reference file.
4. Apply behavior fixes before visual polish.
5. Pair with `pmtl-review-web-ui` if the task is review-first.

## Read only what applies

- `references/forms-and-errors.md`
- `references/navigation-and-focus.md`
- `references/state-feedback.md`

## PMTL interaction tone

- calm
- direct
- respectful
- keyboard-complete
- accessible without theatrics
