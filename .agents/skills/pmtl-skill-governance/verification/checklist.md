# Skill Verification Checklist

- The skill maps to one taxonomy slot in `docs/architecture/skills-taxonomy.md`.
- `SKILL.md` states purpose, trigger, execution, and verification explicitly.
- The folder contains reusable assets when the workflow is non-trivial.
- The skill has a defined pass/fail check, not only advice prose.
- `gotchas.md` or an equivalent evolution note captures recurring failure modes.
- `changelog.md` records meaningful updates so behavior drift is visible.
- Legacy or compatibility skills are marked as such instead of pretending to be canonical.
