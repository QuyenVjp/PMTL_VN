# PMTL Workflow Router Verification Checklist

- The task is mapped to one canonical PMTL skill before any generic skill is chosen.
- A Superpowers skill is added only when the task needs workflow discipline, not to replace PMTL repo policy.
- A global skill is added only when external tooling or platform-specific mechanics are needed.
- The chosen path ends with an explicit verification step.
- Legacy compatibility skills are not the default unless the user explicitly asks for one.
